React学习笔记(一)

一.概念

-初期用于架设instagram
-library:小而巧 优点：方便从一个库切换到另外的库 代码不容易改变
-Framework:大而全 提供整套解决方案 切换框架比较困难

与vue对比

1.组件化
模块化是从代码分析
把可复用的代码抽离为单独的模块便于项目维护和开发

组件化是从界面分析
把可复用的ui元素抽离为单独的组件便于项目维护和开发

组件化的好处是随着项目规模增大，手里组件越来越多，很方便就能把现有的组件拼接为完整的页面

vue如何实现组件化
通过.vue文件来创建对应的组件 .vue分为
-template 结构
-script 行为
-style 样式

react如何实现组件化
react有组件话概念但没有vue这样的模板文件
react中一切都是由js来表现的 需掌握react.js以及ES6 ES7(async和await)

vue:阿里系
react:Facebook qq 京东 百度

2.核心概念

(1)虚拟DOM
-DOM本质：浏览器中的概念 浏览器提供 用js对象表示页面上的元素 并提供了操作DOM对象的API
-React中的虚拟DOM：是框架中的概念 程序员提供 用JS对象来模拟页面上的DOM和DOM嵌套
-为什么要实现虚拟DOM：为了实现页面中DOM元素的高效更新 (按需渲染页面)

-DOM树的概念：一个网页呈现的过程
(1)浏览器请求服务器获取页面的HTML代码
(2)浏览器在内存中解析DOM结构并在浏览器内存中渲染出一棵DOM树
(3)浏览器把DOM树呈现到页面上

-如何实现页面的按需更新：获取内存中的新旧两棵DOM树 进行对比并得到需要更新的DOM元素
-如何获取新旧DOM树：浏览器中没有直接提供获取的API 要手动模拟新旧两棵DOM树

总结
虚拟DOM是以JS对象存在 来模拟页面上DOM嵌套关系

(2)Diff算法
-tree diff:即新旧两棵DOM树逐层对比的过程 当整棵树逐层对比完毕 则所有需要被按需更新的元素必然能够找到
-component diff:在进行tree diff的时候每层中组件级别的对比
如果对比前后组件类型先跟她则暂时认为这个组件不需更新
对比前后 组件类型不同 则需要移除旧的组件 创建新组件并追加到新组件上
-element diff:在进行组件对比的时候 如果两个组件类型相同则需要进行元素级别的对比 这叫做element diff

二.在项目中使用React

1.创建一个基本的webpack项目
(1)创建一个新的文件夹 打开 并运行npm init -y快速初始化项目
(2)在项目里创建dist 和 src 目录
(3)在src下新建一个index.html的文件 用!+tab创建标准html页面
(4)安装webpack 运行 cnpm i webpack -D与cnpm i webpack-cli -D
在webpack 4.x中有很大的特性 就是约定大于配置 打包的入口路径是src -> index.js
(5)在src文件夹下创建一个index.js 且在最外层新建一个文件名为webpack.config.js
(6)在终端输入webpack 便自动在dist下生成一个main.js文件
(7)注意：webpack4.x提供了约定大鱼配置的概念 目的是为了尽量减少配置文件的体积
-默认约定了
-打包的入口是src->index.js
-打包的输出文件是dist->main.js
-4.x新增了mode选项 development和production
-Node支持 Chrome支持的特性 不支持export default
(8)安装cnpm i webpack-dev-server -D 为的是实时更新代码时能随时自动编译运行 在package.json里的script里面添加”dev”:”webpack-dev-server” 在终端里输入cnpm run dev 获得本地端口 在浏览器打开 得到根目录 但是看不到打包好的main.js
webpack-dev-server将打包好的main.js托管到内存中 所以在根目录中看不到 但是可以认为在根目录中都有一个看不见的main.js
(9)可以在dev中设定 自动打开(浏览器) –open safari(chrome) 端口–port 3000 设定localhost –host 127.0.0.1
(10)cnpm i html-webpack-plugin -D 安装此插件 在config.js引入插件将页面生成到内存中去

踩坑时间
安装好plugin后记得在config.js的module.exports里新增一行
plugins: [htmlPlugin]
让打开的默认主页不为根目录而是 主页index.html的内容
踩坑结束
