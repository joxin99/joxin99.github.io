Node.js学习笔记(三)

一.Node中的模块系统

1.使用Node编写应用程序主要就是在使用

(1)EcmaScript语言
和浏览器不一样 Node中没有BOM 和 DOM
(2)核心模块
文件操作fs
http服务的http
url路径操作
path路径处理
os操作系统信息

核心模块的本质也是文件
核心模块文件已经被编译到了二进制文件中了，我们只需要按照名字来加载就可以了

(3)第三方模块
如art-template
凡是第三方模块都必须通过 npm 来下载
使用的时候就可以通过 require(‘包名’) 的方式来进行加载才可以使用
不可能有任何一个第三方包和核心模块的名字是一样的
既不是核心模块、也不是路径形式的模块
先找到当前文件所处目录中的 node_modules 目录
a.node_modules/art-template
b.node_modules/art-template/package.json 文件
c.node_modules/art-template/package.json 文件中的 main 属性
d.main 属性中就记录了 art-template 的入口模块
e.然后加载使用这个第三方包
f.实际上最终加载的还是文件

如果 package.json 文件不存在或者 main 指定的入口模块是也没有
则 node 会自动找该目录下的 index.js,也就是说 index.js 会作为一个默认备选项

如果以上所有任何一个条件都不成立，则会进入上一级目录中的 node_modules 目录查找
如果上一级还没有，则继续往上上一级查找
如果直到当前磁盘根目录还找不到，最后报错：can not find module xxx

注意：我们一个项目有且只有一个 node_modules，放在项目根目录中，这样的话项目中所有的子目录中的代码都可以加载到第三方包
不会出现有多个 node_modules

npm——node package manager
package.json建议每个项目都要有一个package.json文件 可以通过npm init 创建 每次安装包的时候都加上–save保存在package.json里
通过npm i会将node_modules文件都安装回来

优先从缓存加载:由于 在 a 中已经加载过 b 了,所以这里不会重复加载. 可以拿到其中的接口对象，但是不会重复执行里面的代码.这样做的目的是为了避免重复加载，提高模块加载效率
(2)导出exports
Node是模块作用域 默认文件中所有成员只在当前文件模块中有效
对于希望可以把被其他模块访问的成员必须将其挂载到exports接口对象中
