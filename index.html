Node.js学习笔记(一)

1.入门

-为什么学Node.js
-Node.js是一个服务器(服务端)
-利用javascript进行编程
-js运行时环境 解析执行js代码 以前需要浏览器才能运行 现在可以脱离
-理解以便于配合服务端人员进行协同开发

-Node.js中的javascript
-没有DOM BOM
-EcmaScript
-在Node这个环境中为JS提供一些服务器级别的操作API
如：文件读写
网络服务的构建
网络通信
http服务器…等

-Node.js特性
-event-driven事件驱动
-non-bocking I/O model 非阻塞IO模型
-lightweight and efficient

-Node.js包管理工具——npm最大的开源库
-绝大多数的js相关包都存放在npm上 方便使用

-Node.js构建于Chrome的V8引擎(最快最高效)上
-Node.js作者将Chrome的V8引擎移植开发一个独立的js运行环境

-Node.js能做什么
-Web服务器后台
-命令行工具(主要是第三方开发 webpack、gulp、npm)
npm(node)
hexo(node)
git(c)
-模块化编程
@import(‘文件路径’)
Node中可以用@import引用加载JS脚本文件
-异步编程
回调函数
Promise
async
generator

-Node环境安装
-打开终端 查看node版本 输入node -v
-查找node安装位置 输入which node
-安装最新版本13.9.0 输入sudo n 13.9.0 并输入权限密码

以下是踩坑时间
但是我发现我的node最新版本安装在了/usr/local/bin/node下 但是依旧采用了另一个路径下的旧版本 搜不到解决方案于是我决定删除旧版本 rm -rf /旧版本路径
-然而删除以后还是没办法使用上新版本 输入n 只会显示在旧路径下找不到文件 并且重新打开终端以后 出现以下的字段

于是我百度了一下，参考这一个https://www.cnblogs.com/xinpureZhu/p/6088133.html
发现用 nvm ls 查到的都是node版本号 因此 输入 nvm install 13.9.0 居然安装node成功了
成功标志为：
1.在终端输入n 也能返回版本号13.9.0
2.输入node -v返回版本号13.9.0
3.输入which node 返回/usr/local/bin/node
不过以上代码区的问题仍然存在
(此处先收藏一个nvm的命令行说明网址：https://www.cnblogs.com/gaozejie/p/10689742.html)

继续百度 发现了https://blog.csdn.net/A_LWIEUI_Learn/article/details/88653493 其中有一个命令行说：npm install -g nvm 让我意识到可能删除node旧版本的文件时把nvm也给删了

-继续安装稳定版本的node 输入 nvm stable
-顺利安装 没有问题 也不再提示 “nvm install N/A”
nvm和n都是node版本管理工具

接着我发现 随着node的重装 我的hexo命令也没法用了 只好重新安了一遍 步骤如下：

=打开我博客的文件夹 cd myblog
=在文件夹下 npm install
于是hexo又能正常使用了
结束踩坑时间 继续nodejs学习

-读写操作文件
必须引进fs核心模块 fs包括文件操作API 如fs.readFile读取文件
1.使用require加载fs核心模块

2.读取文件 第一个参数为读取的文件路径 第二个是一个回调函数
包括
(1)error 读取失败则error是错误对象 读取成功error则是null
(2)data 读取失败则data是undefined 读取成功data则是读取的数据

但是包含fs模块的js文件如果在浏览器用script标签引入 那么会报错 因为是在node命令行执行的模块
3.写文件
(1)第一个参数：文件路径 第二个参数：文件内容 第三个参数：回调函数
(2)error:通过error判断错误是否发生
文件写入成功error是null 文件写入失败error是错误对象

-http模块 构建服务器
1.加载http核心模块
2.使用http.createServer()创建一个web服务器实例
3.服务器： 代码如果被修改应该重启服务器才能生效
为数据提供服务：发请求接请求处理请求给反馈 注册request请求时间 当客户端请求过来会自动出发服务器的request请求事件然后执行第二个参数 回调处理

5.服务器对象
请求对象：request
响应对象：response
所有请求路径都是以/开始的

则返回内容为：”收到客户请求,请求路径是：/a”
响应对象只能是二进制数据 或者字符串
当代码里面有end()时 说明整个程序执行完了 于是会在浏览器页面输出hello和nodejs 如果没有end() 那么服务器会一直处于待响应状态 不会输出内容

3.其他相关知识

-ip地址和端口号
ip用来定位计算机 端口用来定位具体的应用程序
(所有需要联网通信的软件必须具有端口号 0~65536)
两个js文件里 分别开两个服务一个占用端口3000另一个占用5000
开启两个终端 在浏览器里分别打开 互不影响
编码内容
在服务器端默认发送的数据是utf8编码的内容
但是浏览器并不知是utf8并按照默认的编码去解析
中文操作系统默认为gbk
因此要告诉浏览器发送的内容是什么编码

加上前一句后 在浏览器network里面能查看到response headers里面多了一个content type项
text/plain 普通文本格式
text/html html格式
image/jpeg 图片格式
常用资源对照表参考https://tool.oschina.net/commons 一个url对应到一个资源 由自己定义
另外 如果js文件读取html文件让它在浏览器中显示页面内容 一旦html内容修改 只需刷新浏览器 不用重启服务 即能更新页面
